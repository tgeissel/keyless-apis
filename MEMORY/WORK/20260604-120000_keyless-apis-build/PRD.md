## Context

Build a live-tested, machine-readable catalog of public APIs that (a) require no API key and (b) allow CORS for browser calls. The differentiator vs. existing lists (public-apis etc.) is that the repo tests itself via CI and outputs a verified `apis.json` + `status.json`.

The user wants TypeScript/Node (Playwright ecosystem), ~30 seed APIs, hybrid CORS evaluation, weekly GitHub Actions CI, and a README with a live badge + auto-generated table. No key-required APIs, no SaaS layer, no hand-maintained tables.

### Risks
- CORS is browser-only; server-side fetch gives false positives — must use Playwright for real CORS verification
- Rate limits on free APIs (Nominatim, Genderize etc.) can cause false negatives in CI
- Some "keyless" seeds (MealDB test key "1") are debatable — treat as keyless for now
- GitHub Actions Playwright setup adds complexity (must install browsers)
- status.json committed by CI requires write permissions (PAT or permissions: write)

### Plan
1. Repo structure + package.json (TypeScript, Playwright, tsx)
2. apis.json schema + seed data (30 entries)
3. Checker: header-inspect preflight + Playwright CORS verify hybrid
4. status.json writer + README table generator
5. GitHub Action (weekly + PR trigger)
6. README + CONTRIBUTING
